# 绘本章节分镜切分调优记录（2026-07-20）

本文记录「最近文章总是 12 张分镜」排查与 `picture_book_chapter_scene_plan_v2` prompt 多轮调优过程、踩坑与结论。实现入口：`PictureBookService._chapterPlanPromptRuleLines` / `_chapterPlanPromptTurns`。产品规则摘要见 `AGENTS.md` 与 `docs/ai-call-flow-and-prompt-logic.md`。

## 1. 问题与初判

**现象**：用户最近提交的两篇文章，章节规划审核里 `scenes.length` 均为 12。

**怀疑**：是否回退到本地「平均分配」占位逻辑。

**结论（代码审查）**：正式 `article.create`（绘本开启）走 AI 章节规划，**不是**均分。

| 路径 | 何时触发 | scene 数来源 |
|------|----------|--------------|
| AI 规划 | 保存时 `generateChapterPlanForArticle` → `summary_json` | 模型返回；硬上限 `<= 12` |
| 占位草稿 | 无有效 `summary_json` 计划时打开审核 | `_draftSceneRanges`：多段超 12 或单段超 12 句时才均分；`sceneDescription` **为空** |

基线样本（运行库）：

| 文章 | id | 句数 | 基线 scenes |
|------|-----|------|-------------|
| E38 - The Court Trial Scene | 98 | 41 | 12 |
| E37 - The Hatter's Testimony | 97 | 43 | 12 |

二者均有完整 AI `sceneDescription`，区间不均匀 → 属 AI 路径顶满上限，非占位均分。

## 2. Prompt 结构与约束

章节规划固定 Rules 分块：

1. **对话→叙事**（已调优，本轮基本不动）：引语/歌词/内心独白转第三人称可见动作；禁引语、禁 say 链、禁空洞 meta。
2. **场景切分**（本轮主改）：由「同连续故事场景 + 茶桌/厨房举例」演进为通用「插画情况」三轴。
3. **输出/覆盖**：`scenes.length <= 12`、全槽覆盖、隐藏槽并入邻景、`pageIndex = i`。

**产品约束**：规则必须通用，**禁止**为单书/单场景（Alice、法庭、茶桌等）写合并特例。

## 3. 调优迭代与实测

验证方式：Windows Release + QA（`127.0.0.1:39317`），脚本 `tools/qa_chapter_plan_prompt_opt_retest.mjs`，对 E37/E38 原文走真实 `article.create` + `pictureBookEnabled: true`。过程产物与审核草稿在 `output/prompt-opt-retest/`（运行数据，可不入库）。

### 3.1 第一轮：Rules 去重精简（不删语义）

- **做法**：合并同义 bullet（约 34→26 条），叙事/合并约束保留。
- **实测**：E38 仍 12；E37 12→11。
- **评价**：长度略降，切分几乎无质变。

### 3.2 第二轮：通用「插画情况」三轴（删除场景专名）

切分只认三轴是否变化：

1. 地点 / 时间  
2. 主在场焦点人物组（入场/离场且注意力转移）  
3. 中心进行中活动  

编号句子只是**覆盖锚点**，不是切景理由。删除「tea-table / kitchen / roadside」等专名合并句。

- **实测（id 101/102）**：E38 **12**；E37 **8**（入场+日期+陪审合并 span=9）。
- **评价**：单线连续活动（长证词）明显改善；同场多事件串烧仍易顶 12。

### 3.3 第三轮：活动≠节拍 + 误超压 Scenes（回归）

- **意图**：澄清「中心活动 = 主任务（谁在对谁做什么），不是每个可见节拍」；余波收拾不单独开景；并压缩 Scenes 段长度。
- **错误做法**：把多条 Scenes 规则压成 2–3 条超长合并句。
- **实测（id 106/107）**：**严重回归**——前 11 景各约 1 句，剩余全部塞进最后一景（dump）。
- **结论**：**禁止**再把 Scenes 压成「长墙」；分条短规则更稳。

### 3.4 第四轮（落地版 v3b）：短 bullet + 澄清 + 禁 dump

在分条短规则上保留并加强：

- 中心活动 = 主任务 ≠ 每个可见节拍 / 道具 / 姿态 / 台词  
- 同类型重复微动作、同一事故的直接收拾余波不单独开景  
- Hard cap 外明确：**不要**「多张一句再把剩余塞进最后一张」  

实现注释写明 3.3 回归原因（`_chapterPlanPromptRuleLines`）。

- **实测（id 108/109）**：E38 **10**；E37 **6**。无 dump，非均分，描述齐全。

### 3.5 第五轮（复审优化版 v4）：区间忠实度 + 边界终检 + 严格校验

复审发现 v3b 虽然减少了场景数，但 E38 曾把应由 Hatter 执行的动作错写给 Alice；同时解析端会静默丢弃第 13 景及以后内容，再由覆盖归一化把剩余句子扩进最后一景，与 prompt 的「禁止尾部倾倒」目标冲突。

本轮只做短规则增强，不恢复长墙：

- 将「主任务 of the gathering」改为通用的「焦点人物的主任务及目标」，主视觉焦点变化不依赖人物入场/离场。
- 每个 `sceneDescription` 只能使用自身句子区间内的事件，并保持人物动作归属。
- 同一事故的原因、直接结果和立即收拾保持同景。
- 返回前逐对审核相邻边界；三轴无实质变化且一张图可共同表达时合并并重编号。
- 新生成的 AI 计划严格校验 `<= 12`、顺序 pageIndex、连续完整句子覆盖和非空描述；无效响应直接报错，不再静默截断/修补。旧持久化计划仍兼容读取。
- QA 脚本新增连续覆盖、引号台词候选和「一句一景 + 尾部倾倒」检测；自然物件标签（如 `ORANGE MARMALADE`）不误判为台词。大体积 create 产物缩减为文章与审核必要字段。

真实 Windows Release + QA 复测（id 110/111）：

- E38：**7 景**；上一轮 Hatter→Alice 的人物错配已修正；King 交接下一证人与 Alice 被点名合并；Alice 掀翻 jury box 与立即捡回合并为同一景。
- E37：**6 景**；无人物动作错配，长段证词保持连续。
- 两篇均为连续完整覆盖、顺序索引、无空描述、无引号台词、无均分占位、无尾部倾倒；QA integrity failures = 0。
- 剩余瑕疵：E38 第 2 景出现不自然短语 `dramatic royal punishment him`；第 4 景把 cook 作证与 Dormouse 被逐合成一景，略有过度合并。E37 第 0/1 景和第 3/4 景按三轴仍有继续合并余地。说明结构与人物归属显著改善，但可见叙事措辞和「合并过度/不足」仍存在模型波动。

### 3.6 旧文章重复生成：跨体裁效果与模型波动

为避免只用 E37/E38 庭审样本得出过度乐观结论，复用运行库旧文章，通过 `promptReview` + `refreshPromptReview(chapterPlan)` 每篇生成 3 次。该路径不新增文章、不保存分镜、不确认出图。样本：

| 文章 | 结构 | 句子槽 | 3 次 scene 数 | scene 数标准差 | 边界 Jaccard | 波动 |
|------|------|-------:|----------------|----------------:|-------------:|------|
| id 46 `Alice Follows The Rabbit` | 连续动作/移动 | 46 | 4 / 4 / 6 | 0.94 | 0.556 | 中 |
| id 68 `What do I like best about school?` | 非虚构说明文 | 25 | 11 / 7 / 5 | 2.49 | 0.507 | **高** |
| id 91 `The Proud Spring Onion` | 长篇拟人叙事 | 87 | 7 / 9 / 结构无效 | 1.00（成功两次） | 0.556 | 中；1/3 失败 |

结论：

- id 46 的核心动作边界稳定：追兔至洞口结束于 21，落入深井结束于 27，三次一致；第 3 次额外把开头内心考虑、坠落末尾内心感想拆开，说明对内心独白的微节拍拆分仍有随机回归。描述中的 `ORANGE MARMALADE` 是允许的自然标签，不是引号台词；QA 已修正该误报。
- id 68 暴露真正的 prompt 覆盖缺口。说明文没有单一“正在进行的主任务”，模型会在「每个事实一景」与「多个主题做 montage」之间随机选择，导致 11/7/5 的高波动。5 景版本最接近连贯主题块，11 景版本明显过碎。
- id 91 的稳定核心边界为 6/12/24/56/76；7 景版本把刀压、脏水、丢弃合成 26 句长景，9 景版本拆为更合理的动作阶段。第 3 次被严格校验拒绝，证明严格失败机制必要，也量化出本组 9 次远程规划中 **1 次结构无效（11.1%）**。
- v4 对叙事类文章的主要边界有效，但边缘场景仍是中等波动；对说明文则不稳定。E37/E38 的改善不能直接外推为全体裁稳定。

### 3.7 v5：局部事实块规则与三轮淘汰复测

目标是修复 id 68 的说明文高波动，同时守住 id 46/91 的叙事边界，并用 id 69 的三明治制作步骤检查是否误伤流程动作。每个候选版本均构建真实 Windows Release 后，通过审核草稿刷新重复生成；不新增文章、不保存计划、不确认出图、不调用图片 API。

先后测试了三种写法：

1. **宽泛体裁分支（淘汰）**：把说明、描述、指令类都纳入主题/时间/流程块。id 68 收敛到 7/7/8，但 id 46 出现一次 9 景过拆，说明 `descriptive` 范围过宽。
2. **收窄体裁分支（淘汰）**：仅在“主要是说明文且没有连续人物动作”时启用，并排除叙事/流程。模型仍需先分类范围，id 68 回到 9/6/7，id 46 两次变为 3 景并把追兔与入洞合并，id 91 也升为高波动。
3. **局部事实块规则（保留）**：不判断整篇体裁，只识别同一主题与时间/地点框架内的连续事实、例子、列表项或一般陈述；这些内容共用一张 montage。明确该规则不能覆盖顺序移动、物件操作、发现、事故或其它因果动作。

四篇文章的版本对照：

| 版本 | id 46 连续叙事 | id 68 说明文 | id 69 流程文 | id 91 长篇叙事 | 结构失败 |
|------|----------------|--------------|--------------|----------------|----------|
| v4 基线 | 4 / 4 / 6 | 11 / 7 / 5 | 3 / 3 / 3 | 7 / 9 / 无效 | 1 / 12 |
| 宽泛体裁分支（淘汰） | 4 / 9 / 4 | 7 / 7 / 8 | 3 / 3 / 3 | 9 / 10 / 9 | 0 / 12 |
| 收窄体裁分支（淘汰） | 3 / 3 / 4 | 9 / 6 / 7 | 3 / 3 / 3 | 10 / 9 / 8 | 0 / 12 |
| **v5 局部事实块（保留）** | **4 / 4 / 4** | **8 / 8 / 9** | **3 / 3 / 3** | **9 / 9 / 9** | **0 / 12** |

最终候选的波动指标：

| 文章 | scene 数标准差 | 边界 Jaccard | 波动 | 语义复核 |
|------|----------------:|-------------:|------|----------|
| id 46 | 0 | 1.000 | 低 | 三次均为 0–8、9–21、22–27、28–45；追兔、入洞、下坠边界正确 |
| id 68 | 0.47 | 0.583 | 中 | 学校背景、日程、科目、活动/校务、朋友等主块稳定；活动/校务内部仍有 8/9 景粒度差异 |
| id 69 | 0 | 1.000 | 低 | 三次均在 2、7 切分；介绍/配料、制作、成品阶段未被 montage 规则吞并 |
| id 91 | 0 | 0.733 | 低 | scene 数稳定为 9；核心边界 6/12/24/30/56/76 全部重复命中，中段仅有 39/40、65/68 小幅漂移 |

相对 v4，id 68 的 scene 数标准差由 2.49 降至 0.47，范围由 6 降至 1；改善重点是**一致性**，不是追求更少图片。最终 12 次响应全部通过连续覆盖、顺序索引、非空描述与 scene 上限检查；QA 汇总为 0 个引号台词候选、0 个 speech-process 词、0 个畸形安全替换短语，但仍有 4 个非可见思考措辞候选（如 `wonders` / `reflecting`）。这些措辞不影响本轮边界结论，但说明对话/内心独白转可见动作仍未完全稳定。由于每篇只有 3 次随机采样，不能据此宣称模型波动已经消失；id 68 仍保留中等边界波动。

## 5. 合理性结论（按三轴审核，非按数量）

- **E37（6）**：Alice 长大侧线与 Hatter 作证主线区分清楚，但 Hatter 的入场/帽子/紧张仍可进一步合并，第 3 景也可并回后续证词；约 **B+**。  
- **E38（7）**：人物归属错误已修复，掀翻/捡回和程序性交接均合并到位；cook→Dormouse 段略有过度合并，另有一处不自然英文；约 **A− / B+**。  

结构质量已从「数量改善」提升到连续覆盖、无 dump、区间人物归属正确；但对话转可见动作仍会偶发退化为 `notes inwardly`、`asks`、`answers` 等叙述词，不能表述为“描述质量全程良好”。

v5 旧文章矩阵表明：局部事实块规则把说明文 scene 数波动从高降到中，同时没有破坏流程动作和两篇叙事回归。它没有把 id 68 强行压到固定数量，活动/校务内部仍会因“同一 umbrella topic 能否共用一张 montage”产生边界差异。

## 6. 最终规则口径（勿回退）

1. 一张绘本页 = 听力区间共用插画，不是文学微节拍列表。  
2. 仅当三轴至少一轴实质变化时开新 scene。  
3. 中心活动 = 这场戏的**主任务**，不是每个新动作/台词。  
4. 不因对话轮次、反应、情绪、旁白、同类型重复微动作、事故直接余波单独拆。  
5. 编号句是覆盖锚点；一张图可覆盖很多句。  
6. 同一主题和时间/地点框架内的连续事实、例子、列表项或一般陈述合成一个事实块 montage，不得一事实一景；该规则不覆盖顺序动作和因果事件。  
7. `scenes.length <= 12` 是上限，不是目标；禁止为凑满上限而拆，禁止一句一景再 dump。  
8. 不要写死书名或场景类型名当合并条件，也不要让模型先判断整篇体裁。  
9. 每个分镜只使用自身句子区间内的事件，保持谁做了什么与原文一致；返回前逐对审核相邻边界。  
10. 新生成的 AI 计划结构不合法时直接报错，不静默截断或把余量扩进最后一景；旧 `summary_json` 不自动迁移，需刷新规划或重新创建文章才用新规则。

## 7. 还有没有优化空间？

**仍有优化空间，但本轮事实块缺口已经得到有效改善，暂不应继续围绕“场景数更少”单点调词。**

v4 已解决本轮最明确的结构问题。剩余重点是：

- 对话/内心独白仍偶发写成 `notes inwardly`、`asks`、`answers`，没有完全转成可见动作。
- 「同一张图能否共同表达」可能导致过度合并（E38 cook 作证 + Dormouse 被逐），而同一作证活动也仍可能拆成两景（E37）。
- v5 最终矩阵中三篇回归为低波动，id 68 仍为中等波动；学校活动、旅行、校务与家长会是否属于同一 umbrella topic 仍有合理但不完全一致的边界判断。
- 本轮 12 次没有再出现结构无效响应，但 v4 曾有真实无效输出，严格校验仍必须保留，不能据 12 次样本移除。

不建议：

- 再压 Scenes 成超长句  
- 数字目标（如 4–9 张）  
- 场景专名补丁  
- 本地后处理强制合并 scene  

产品上：保留 v5 局部事实块规则。它比体裁分支更可控，对 id 68 有实质改善，且 id 46/69/91 回归稳定。后续若继续调优，应先积累第二篇更长的说明文并至少重复 5 次；不要继续只用 id 68 追加 prompt 句子，否则容易过拟合。E37/E38 也不再用于追求更少 scene，只保留为对话/人物动作归属回归。
